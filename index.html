Module Overview

Welcome to Data Mining,
We will start this module with a warm up lab to set up software and some basic data analysis. The formal lecture will give an overview of the module — what will be covered, how it will be delivered and assessed. In addition, background concepts in statistics and optimisation are discussed.

Exploratory Data Analysis

Before we begin to think about constructing models representing our data we need to see what kind of data we have, what is being measured, first steps to clean data. etc.
  • Data and metadata
  • Statistics for understanding
  • Features from data

Data Modelling - Introduction

This week we will discuss general concepts/issues in the construction of data mining models.
  • Type of models
  • Model building as a process
  • Modelling concerns

Regression

Sometimes we need to predict a numeric value or set of such values, given existing (training) data
  • Motivation - fitting a line to data
  • Perspectives - optimisation, linear algebra, statistics
  • Measuring the quality of the results
  • Regularisation
  • Transformation of variables
  • Regression in practice

Classification

In classification we are interested in predicting the class of given data observations, i.e., the target is discrete/categorical and not continuous as in regression.
  • Links with regression
  • Evaluation of classification models

Data Modelling - Advanced

This week we will look at model building process in more detail, in particular feature engineering/selection, hyperparameter tuning and combining models to generate ensemble Learning. Finally we will also look a explainable AI (XAI).

Advanced Classification

Classification using techniques that are more interpretable, offer high accuracy and/or work with very high dimensional data
  • Decision Trees
  • Ensemble classifiers
  • Support vector machines

Clustering

Given unlabeled data, look for subsets that help to improve understanding of the overall data set
  • Partitioning
  • Hierarchies

Association Analysis

Association Rule Analysis (ARM) or Market Basket Analysis is concerned with the generation of rule linking items that tend to appear together.
  • Measures of association
  • Algorithms: Apriori

Anomaly Detection and Recommendation Systems

  • Anomaly detection algorithms
  • Item-based recommendation
  • Collaborative Filtering
