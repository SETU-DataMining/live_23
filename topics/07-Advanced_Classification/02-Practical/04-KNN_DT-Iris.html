Predicting Iris species using KNN and Decision Trees

We have already considered how to use K-nearest-neighbors (KNN) to predict Iris species.

Here we add the Decision Tree approach that was shown in class, so you can see how some of the analysis and outputs shown in the notes were generated.

The dataset is available in sklearn but the notebook for this practical also uses some supporting python code that can be found in this zip archive.

That needs to be unzipped in the same directory as where you saved the notebook.

It might also be worth checking that you have already installed the pydotplus package, from the conda-forge channel or whatever.

When you run the notebook, you should see that it is relatively easy, because of the conventions used in sklearn, to switch between two such vastly different classifiers.

Hence, as an exercise, you could try some hyperparameter tuning. The dataset is quite small so even GridSearchCV should run relatively quickly.
